# UI Standards

## Principles

- Build the usable experience first, not a marketing shell unless requested.
- Match the interface to the domain and user workflow.
- Keep navigation predictable.
- Avoid decorative complexity that harms clarity.
- Design for keyboard and assistive technology use from the start.

## Accessibility

- Target WCAG 2.2 AA by default.
- Ensure visible focus states.
- Support keyboard operation.
- Use semantic elements.
- Provide labels, names, and descriptions for controls.
- Do not rely on color alone.

## Layout

- Use responsive constraints for fixed-format UI elements.
- Avoid overlapping text and controls.
- Ensure text fits within its container.
- Keep repeated cards and panels visually consistent.

## Validation

- Test across desktop and mobile sizes.
- Run automated accessibility checks.
- Perform keyboard navigation checks.
- Spot-check screen reader behavior for key flows.
