# Definition Of Done

An increment is done when:

- [ ] Requirements and acceptance criteria are satisfied.
- [ ] Implementation aligns with approved architecture.
- [ ] Tests validate meaningful behavior.
- [ ] Tests pass locally or in CI.
- [ ] Security risks are reviewed.
- [ ] Accessibility impact is reviewed.
- [ ] Observability impact is reviewed.
- [ ] Documentation is updated.
- [ ] Technical debt is documented.
- [ ] Release or rollback notes are updated where applicable.
- [ ] Human milestone gate approval has been recorded.
