# Harness Security Notes

The harness promotes secure development practices through:

- security requirements during discovery
- architecture security model review
- threat modeling for sensitive work
- secure coding standards
- dependency and secrets scanning gates
- security review before release

No application secrets, credentials, tokens, or private keys should be stored in this repository.
