# Harness Requirements

This document describes requirements for the harness repository itself.

## Functional Requirements

| ID | Requirement | Status |
| --- | --- | --- |
| HR-FR-001 | Provide a phase-based engineering process for Codex-assisted development. | Draft |
| HR-FR-002 | Provide templates for requirements, architecture, implementation planning, testing, security, accessibility, and release review. | Draft |
| HR-FR-003 | Provide standards and checklists that can be copied into project repositories. | Draft |
| HR-FR-004 | Provide placeholders for reference architectures and sample applications. | Draft |
| HR-FR-005 | Provide Codex-facing operating instructions. | Draft |

## Non-Functional Requirements

| ID | Category | Requirement | Status |
| --- | --- | --- | --- |
| HR-NFR-001 | Maintainability | Assets must be understandable and easy to adapt. | Draft |
| HR-NFR-002 | Traceability | Major process decisions must be documented. | Draft |
| HR-NFR-003 | Security | Harness guidance must promote secure development practices. | Draft |
| HR-NFR-004 | Accessibility | Harness guidance must promote accessibility-by-default. | Draft |
| HR-NFR-005 | Portability | Assets should be usable across many project types. | Draft |

## Acceptance Criteria

- The harness can be copied into a new project.
- Codex can read `AGENTS.md` and understand required gates.
- Templates cover all major phases.
- Standards and checklists are available before implementation starts.
