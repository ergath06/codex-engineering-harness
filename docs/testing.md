# Harness Testing Notes

The harness expects testing to validate real behavior, not superficial success.

Recommended test coverage includes:

- unit tests
- integration tests
- end-to-end tests
- accessibility checks
- security validations
- performance checks where required
- failure-mode tests
- regression coverage
